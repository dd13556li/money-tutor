## 二、單元特色

### 2.1 教學定位

B5 訓練「在預算限制下，滿足必要條件後進行最佳選購」的複合預算能力。
學習路徑：**加總計算（B1/B2）→ 比較判斷（B4）→ 限制條件下選購（B5）→ 市場採購（B6）**

### 2.2 必買約束系統（B5 核心特色）

B5 設計了**必買 vs. 選購**的雙層商品結構：

| 商品類型 | 圖示 | 行為 | 教學意義 |
|---------|------|------|---------|
| **必買** | 🔒 鎖定標記 | 初始已選中，無法取消 | 固定支出概念 |
| **選購** | 可自由選取 | 點擊切換選取狀態 | 可變支出決策 |

**必買商品**：生日蛋糕 🎂（380元）、果汁飲料 🧃（120元）— 合計 500 元固定支出

### 2.3 即時預算條（`b5-total-bar`）

動態顯示目前選購金額與預算的關係：

| 狀態 | 條件 | 顏色 | 說明 |
|------|------|------|------|
| `ok` | 總金額 ≤ 90% 預算 | 綠色 | 安全範圍 |
| `near` | 總金額 > 90% 預算 | 黃色 | 接近預算上限 |
| `over` | 總金額 > 預算 | 紅色 | 超支警告 |

**教學設計**：黃色「警戒區」讓學生感受接近限制的壓力，培養預算意識。

### 2.5 關卡制結構

**32 組關卡**（easy/normal 各 12 組，hard 8 組；2026-03-24 擴充 easy/normal 各 8→12），每關定義：
- `budget`：本關總預算
- `availableIds`：本關可選商品清單（子集，不含全部 12 件）

### 2.6 智慧移除建議（超支回饋）—— 2026-03-25 新增

超支時，系統自動計算最佳移除建議：在已選的**選購商品**中，找出**最貴且移除後總金額 ≤ 預算**的單一商品，以高亮提示顯示：

```
💸 超出預算了！請重新選擇
  超出 95 元
💡 試試取消勾選「📸 拍立得相機」（150元）！
```

**演算法**（貪婪單步）：

```javascript
const selectedOptionals = g.items
    .filter(i => !i.must && g.selectedIds.has(i.id))
    .sort((a, b) => b.price - a.price);          // 由貴到便宜排序
const suggestion = selectedOptionals
    .find(i => total - i.price <= g.budget);      // 第一個移除後達標的
```

**設計決策**：
- 僅建議**單一**移除（不做多步組合）：簡化認知負擔，符合低年級使用者需求
- 必買商品不進入候選清單（不可移除）
- 若無單一商品能解決超支，不顯示提示（`suggestion` 為 undefined）
- `.b5-removal-hint`（CSS）：淡黃底、琥珀邊框，與 B2/B3/B4 錯答提示風格一致

**教學設計對齊**：源自 B5 超支 → A1/A4 找零步驟提示的「回饋引導」模式，提供具體行動建議而非單純否定。

### 2.7 預算提示鈕（`_showBudgetHint`）—— 2026-03-25 新增

選購畫面加入「💡 還能選什麼？」按鈕（黃框，always visible）：

| 行為 | 說明 |
|------|------|
| 計算剩餘預算 | `budget - getTotal()` |
| 高亮可選商品 | 篩選 `!must && !selected && price ≤ remaining` → `.b5-hint-glow`（金色脈動光暈 × 2 次）|
| 語音提示 | 「還剩 X 元，可以加選 Y 或 Z」|
| 無可選時 | 「還剩 X 元，沒有可以加選的商品了」|

**靈感來源**：B1 `_showCoinHint`（高亮可用錢幣）模式，從「提示付款組合」延伸為「提示可選商品」。

**技術細節**：
- 提示按鈕插入 `b5-result-area` 前、確認按鈕上方
- `g.submitted` 守衛（結果已送出後不觸發）
- `TimerManager.setTimeout` 2500ms 自動移除 `.b5-hint-glow`

### 2.8 關卡轉場卡（`_showRoundTransition`）—— 2026-03-25 新增

`nextRound()` 不再直接呼叫 `renderRound()`，而是先顯示全屏**轉場覆蓋卡**，1.1s 後淡出並進入下一關。

**視覺設計**：
```
╔══════════════════════╗
│   準備好了嗎？         │
│   第 3 關             │  ← 白色大字（3rem）
│   🎂                  │
╚══════════════════════╝
```
- 紫色半透明背景（`rgba(99,102,241,0.88)`）
- 卡片彈出動畫：`b5RtIn`（0.4s，scale 0.75→1.06→1）
- 1.1s 後加 `.b5-rt-fade` class（`transition: opacity 0.3s`）淡出，300ms 後移除並呼叫 callback

**靈感來源**：C6「進入第X關」過渡語音 + `nextQuestionScheduled` 節奏設計 — 防止關卡之間切換過於突兀，給學生短暫的心理準備時間。

**技術細節**：
- 使用 callback pattern：`_showRoundTransition(roundNum, callback)`，`callback = () => this.renderRound()`
- 附加至 `document.body`（覆蓋整個視窗）
- 兩層 `TimerManager.setTimeout`：第一層 1100ms 觸發淡出；第二層 300ms 後移除並觸發 callback
- 兩層均使用 `'turnTransition'` 類別，切換時自動清除

## 四、測驗流程

```
[設定] 難度 × 關卡數 × 重試模式
    ↓
[關卡畫面] renderRound()
    ├── 顯示預算金額（`b5-budget-display`）
    ├── 預選必買商品（locked，selectedIds 預填）
    ├── 渲染可選商品格（點擊切換選取）
    └── 即時更新預算條（ok / near / over）
         ↓
[確認] 按下「確認購買」
    ├── 驗證：mustOk && budgetOk && total > 0
    ├─ 通過 → 語音「太棒了！」→ 顯示「下一關」按鈕
    └─ 失敗（未含必買 or 超支）→ 顯示錯誤說明 + 「再試一次」+ 灰色「跳過」
         ↓
[下一關 or 跳過] → 重複至所有關卡完成 → showResults()
```

## 七、技術注意事項

- 確認邏輯三條件缺一不可：`mustOk`（含必買）、`budgetOk`（不超支）、`total > 0`（有選商品）
- 必買商品在 `renderRound()` 初始化時預填入 `selectedIds`（Set）
- 即時預算條更新觸發於每次點擊商品後（`_updateBudgetBar()`）
- 「再試一次」重渲染同一關卡（`renderRound(currentRound)`），不推進關卡計數
- 「跳過」推進關卡但不計分（不加入 `correctCount`）

## 九、跨系列特色分析（2026-03-24）

### 9.1 B5 vs C5 夠不夠（同為「預算判斷」類型）

| 特性 | B5 生日派對預算 | C5 夠不夠 |
|------|--------------|---------|
| 核心問題 | 「在預算內選什麼？」| 「我的錢夠不夠買這個？」|
| 商品池 | 固定 12 件（2 必買+10 選購）| 48+ 件（動態生成）|
| 金額策略 | 固定場景（B5_SCENARIOS）| PriceStrategy 動態生成 |
| 難度模態 | 商品數量 + 預算緊張度 | 呈現方式（顯示/隱藏金額）|
| 必買限制 | 有（B5 獨有設計）| 無 |
| 圖片資源 | emoji | emoji + 圖片（C5 有圖示）|

### 9.2 B5 vs A3 麥當勞（同為「多選購物」類型）

| 特性 | B5 生日派對預算 | A3 麥當勞 |
|------|--------------|---------|
| 選購限制 | 預算上限（不可超支）| 指定組合（需符合任務）|
| 必選項目 | 2 件必買（蛋糕+飲料）| 依任務指定（套餐組合）|
| 商品展示 | emoji 圖示 + 名稱 + 價格 | 真實餐點圖片 |
| 自訂商品 | 無 | 有（魔法餐點上傳）|
| 付款環節 | 無（只選購）| 有（付款 → 找零完整流程）|

## 十、觸控與桌面支援（2026-03-27 補充）

### 10.1 互動模式與觸控

B5 以「點選商品卡片（購物清單）→ 確認」為主要互動，無拖曳操作。

| 功能 | 桌面 | 行動 |
|------|------|------|
| 點選商品 | 點擊卡片 toggle（選/取消）| 觸控點擊（≥ 44px 卡片）|
| 確認按鈕 | 點擊 `.b5-confirm-btn` | 觸控，按鈕在畫面底部 |
| 建議移除高亮 | 超支後動畫脈動 | 同桌面（CSS animation）|
| 預算提示鈕 | 點擊 💡 | 觸控 |
| 關卡轉場 | 自動（1.1s 動畫）| 同桌面 |

**B5 特殊觸控考量**：
- 商品卡片 `min-height: 64px` 確保觸控容易
- 必買商品（蛋糕/飲料）鎖定不可取消（`pointer-events: none` 遮罩）
- 超支後建議移除：`.b5-hint-glow` 脈動動畫不影響觸控事件

### 10.2 響應式設計斷點

| 斷點 | ≥ 768px | < 768px |
|------|---------|---------|
| 商品格子 | 3 欄 flex-wrap | 2 欄 flex-wrap |
| 商品卡片 | `calc(33% - 10px)` | `calc(50% - 8px)` |
| 預算列 `.b5-budget-bar` | 固定頂部 sticky | 固定頂部 sticky |
| 確認按鈕 | 置中 `max-width: 200px` | 全寬 |
| 關卡轉場卡 | 全屏覆蓋（`.b5-round-intro`）| 同桌面 |

### 10.3 跨裝置測試重點

- **iOS Safari**：全屏覆蓋卡 `position: fixed; top: 0; left: 0; width: 100vw; height: 100vh` 需注意 Safari 的 `100vh` 包含工具列問題 → 改用 `100dvh` 或 JS 計算
- **Android Chrome**：商品格子換行確認不會截斷最後一列
- **平板**：3 欄格子在 `768px~1024px` 過渡期需確認視覺均衡

## 十三、確認按鈕就緒脈動（Round 21，2026-03-28）

### 13.1 功能說明

當派對購物車總金額 > 0 且 ≤ 預算時，「確認選擇」按鈕自動進入 `ready` 狀態，顯示綠色脈動光暈，吸引學生點擊確認。

- 超出預算或尚未選任何商品 → 按鈕 disabled（無脈動）
- 金額合法 → `btn.classList.add('ready')`，`b5ConfirmPulse` 動畫啟動

### 13.2 實作

```javascript
// _updateTotalBar() 末尾
const canConfirm = total > 0 && total <= g.budget;
if (btn) {
    btn.disabled = !canConfirm;
    btn.classList.toggle('ready', canConfirm);
}
```

### 13.3 CSS

```css
#b5-confirm-btn.ready { animation: b5ConfirmPulse 1.4s ease-in-out infinite; }
@keyframes b5ConfirmPulse {
    0%,100% { box-shadow: 0 4px 14px rgba(16,185,129,0.4); transform: scale(1); }
    50%     { box-shadow: 0 6px 24px rgba(16,185,129,0.7); transform: scale(1.03); }
}
```
