## 二、單元特色

### 2.1 教學定位

B5 訓練「在預算限制下，滿足必要條件後進行最佳選購」的複合預算能力。
學習路徑：**加總計算（B1/B2）→ 比較判斷（B4）→ 限制條件下選購（B5）→ 市場採購（B6）**

### 2.2 必買約束系統（B5 核心特色）

B5 設計了**必買 vs. 選購**的雙層商品結構：

| 商品類型 | 圖示 | 行為 | 教學意義 |
|---------|------|------|---------|
| **必買** | 🔒 鎖定標記 | 初始已選中，無法取消 | 固定支出概念 |
| **選購** | 可自由選取 | 點擊切換選取狀態 | 可變支出決策 |

**必買商品**：生日蛋糕 🎂（380元）、果汁飲料 🧃（120元）— 合計 500 元固定支出

### 2.3 即時預算條（`b5-total-bar`）

動態顯示目前選購金額與預算的關係：

| 狀態 | 條件 | 顏色 | 說明 |
|------|------|------|------|
| `ok` | 總金額 ≤ 90% 預算 | 綠色 | 安全範圍 |
| `near` | 總金額 > 90% 預算 | 黃色 | 接近預算上限 |
| `over` | 總金額 > 預算 | 紅色 | 超支警告 |

**教學設計**：黃色「警戒區」讓學生感受接近限制的壓力，培養預算意識。

### 2.5 關卡制結構

**32 組關卡**（easy/normal 各 12 組，hard 8 組；2026-03-24 擴充 easy/normal 各 8→12），每關定義：
- `budget`：本關總預算
- `availableIds`：本關可選商品清單（子集，不含全部 12 件）

### 2.6 智慧移除建議（超支回饋）—— 2026-03-25 新增

超支時，系統自動計算最佳移除建議：在已選的**選購商品**中，找出**最貴且移除後總金額 ≤ 預算**的單一商品，以高亮提示顯示：

```
💸 超出預算了！請重新選擇
  超出 95 元
💡 試試取消勾選「📸 拍立得相機」（150元）！
```

**演算法**（貪婪單步）：

```javascript
const selectedOptionals = g.items
    .filter(i => !i.must && g.selectedIds.has(i.id))
    .sort((a, b) => b.price - a.price);          // 由貴到便宜排序
const suggestion = selectedOptionals
    .find(i => total - i.price <= g.budget);      // 第一個移除後達標的
```

**設計決策**：
- 僅建議**單一**移除（不做多步組合）：簡化認知負擔，符合低年級使用者需求
- 必買商品不進入候選清單（不可移除）
- 若無單一商品能解決超支，不顯示提示（`suggestion` 為 undefined）
- `.b5-removal-hint`（CSS）：淡黃底、琥珀邊框，與 B2/B3/B4 錯答提示風格一致

**教學設計對齊**：源自 B5 超支 → A1/A4 找零步驟提示的「回饋引導」模式，提供具體行動建議而非單純否定。

### 2.7 預算提示鈕（`_showBudgetHint`）—— 2026-03-25 新增

選購畫面加入「💡 還能選什麼？」按鈕（黃框，always visible）：

| 行為 | 說明 |
|------|------|
| 計算剩餘預算 | `budget - getTotal()` |
| 高亮可選商品 | 篩選 `!must && !selected && price ≤ remaining` → `.b5-hint-glow`（金色脈動光暈 × 2 次）|
| 語音提示 | 「還剩 X 元，可以加選 Y 或 Z」|
| 無可選時 | 「還剩 X 元，沒有可以加選的商品了」|

**靈感來源**：B1 `_showCoinHint`（高亮可用錢幣）模式，從「提示付款組合」延伸為「提示可選商品」。

**技術細節**：
- 提示按鈕插入 `b5-result-area` 前、確認按鈕上方
- `g.submitted` 守衛（結果已送出後不觸發）
- `TimerManager.setTimeout` 2500ms 自動移除 `.b5-hint-glow`

### 2.8 關卡轉場卡（`_showRoundTransition`）—— 2026-03-25 新增

`nextRound()` 不再直接呼叫 `renderRound()`，而是先顯示全屏**轉場覆蓋卡**，1.1s 後淡出並進入下一關。

**視覺設計**：
```
╔══════════════════════╗
│   準備好了嗎？         │
│   第 3 關             │  ← 白色大字（3rem）
│   🎂                  │
╚══════════════════════╝
```
- 紫色半透明背景（`rgba(99,102,241,0.88)`）
- 卡片彈出動畫：`b5RtIn`（0.4s，scale 0.75→1.06→1）
- 1.1s 後加 `.b5-rt-fade` class（`transition: opacity 0.3s`）淡出，300ms 後移除並呼叫 callback

**靈感來源**：C6「進入第X關」過渡語音 + `nextQuestionScheduled` 節奏設計 — 防止關卡之間切換過於突兀，給學生短暫的心理準備時間。

**技術細節**：
- 使用 callback pattern：`_showRoundTransition(roundNum, callback)`，`callback = () => this.renderRound()`
- 附加至 `document.body`（覆蓋整個視窗）
- 兩層 `TimerManager.setTimeout`：第一層 1100ms 觸發淡出；第二層 300ms 後移除並觸發 callback
- 兩層均使用 `'turnTransition'` 類別，切換時自動清除

## 四、測驗流程

```
[設定] 難度 × 關卡數 × 重試模式
    ↓
[關卡畫面] renderRound()
    ├── 顯示預算金額（`b5-budget-display`）
    ├── 預選必買商品（locked，selectedIds 預填）
    ├── 渲染可選商品格（點擊切換選取）
    └── 即時更新預算條（ok / near / over）
         ↓
[確認] 按下「確認購買」
    ├── 驗證：mustOk && budgetOk && total > 0
    ├─ 通過 → 語音「太棒了！」→ 顯示「下一關」按鈕
    └─ 失敗（未含必買 or 超支）→ 顯示錯誤說明 + 「再試一次」+ 灰色「跳過」
         ↓
[下一關 or 跳過] → 重複至所有關卡完成 → showResults()
```

## 七、技術注意事項

- 確認邏輯三條件缺一不可：`mustOk`（含必買）、`budgetOk`（不超支）、`total > 0`（有選商品）
- 必買商品在 `renderRound()` 初始化時預填入 `selectedIds`（Set）
- 即時預算條更新觸發於每次點擊商品後（`_updateBudgetBar()`）
- 「再試一次」重渲染同一關卡（`renderRound(currentRound)`），不推進關卡計數
- 「跳過」推進關卡但不計分（不加入 `correctCount`）

## 九、跨系列特色分析（2026-03-24）

### 9.1 B5 vs C5 夠不夠（同為「預算判斷」類型）

| 特性 | B5 生日派對預算 | C5 夠不夠 |

| 金額策略 | 固定場景（B5_SCENARIOS）| PriceStrategy 動態生成 |
| 難度模態 | 商品數量 + 預算緊張度 | 呈現方式（顯示/隱藏金額）|

| 特性 | B5 生日派對預算 | A3 麥當勞 |
|------|--------------|---------|

| 必選項目 | 2 件必買（蛋糕+飲料）| 依任務指定（套餐組合）|

## 十、觸控與桌面支援（2026-03-27 補充）

### 10.1 互動模式與觸控

B5 以「點選商品卡片（購物清單）→ 確認」為主要互動，無拖曳操作。

| 功能 | 桌面 | 行動 |
|------|------|------|
| 點選商品 | 點擊卡片 toggle（選/取消）| 觸控點擊（≥ 44px 卡片）|
| 確認按鈕 | 點擊 `.b5-confirm-btn` | 觸控，按鈕在畫面底部 |
| 建議移除高亮 | 超支後動畫脈動 | 同桌面（CSS animation）|
| 預算提示鈕 | 點擊 💡 | 觸控 |
| 關卡轉場 | 自動（1.1s 動畫）| 同桌面 |

**B5 特殊觸控考量**：
- 商品卡片 `min-height: 64px` 確保觸控容易
- 必買商品（蛋糕/飲料）鎖定不可取消（`pointer-events: none` 遮罩）
- 超支後建議移除：`.b5-hint-glow` 脈動動畫不影響觸控事件

### 10.2 響應式設計斷點

| 斷點 | ≥ 768px | < 768px |
|------|---------|---------|
| 商品格子 | 3 欄 flex-wrap | 2 欄 flex-wrap |

| 預算列 `.b5-budget-bar` | 固定頂部 sticky | 固定頂部 sticky |
| 確認按鈕 | 置中 `max-width: 200px` | 全寬 |

## 十四、教學設計詳析（參照 F/C/A 格式，2026-03-29）

### 14.1 教學定位與學習目標

**核心定位**：B5 以「生日派對採購」的情境，訓練學生在固定預算內規劃多商品選購，理解「必需品 vs 自選品」的消費概念，並練習「加總判斷」與「取捨決策」的預算管理能力。

**學習目標層次**：
1. **知識層**：理解「預算」的概念，理解必需品的約束條件
2. **技能層**：多數字累加，即時判斷是否超預算
3. **應用層**：在有限預算內做出最佳組合決策，理解取捨的必要性

**與其他單元的銜接**：
- 前置：C4/C5（付款與金額判斷）→ B1（費用加總）→ B2（收支概念）
- 後續：B6（完整購物流程，含付款找零）

### 14.2 測驗內容設計

**必需品設計理念**：鎖定 2 件必買商品（蛋糕 380元 + 飲料 120元 ＝ 500元），提供固定基礎費用，讓學生專注規劃「剩餘預算如何分配」，降低全開放選擇的複雜度。

**32 個情境設計規則**：
- 簡單（12 關）：預算寬鬆，多件商品都在預算內，測試「能選多少」
- 普通（12 關）：預算中等，需要思考取捨，部分超預算情境
- 困難（8 關）：預算緊繃，可選商品多但預算有限，高頻率超預算

**選擇難度設計**：避免「隨意選幾件剛好合」的巧合，每個情境都需要學生主動計算。

### 14.3 互動方式詳析

**核心互動循環**：
```
開場介紹卡（關卡預算）1.8s
→ 看商品 + 點擊 toggle 選取
→ 預算儀表條即時更新（綠→橘→紅）
→ 確認鈕 ready 脈動（合法時）
→ 確認 → 答對 / 超支
   ↓ 超支：建議移除商品 + 高亮 2.4s
→ 轉場卡（下一關）
```

**超支處理設計哲學**：不直接告訴學生「移除哪件」，而是「建議」（貪婪算法計算最接近答案的移除方案），學生仍可自行決定，保留決策自主性。

**💡 提示鈕設計**：僅在簡單/普通模式顯示，困難模式完全隱藏，強迫學生自主判斷。觸發後高亮「還買得起」的商品（`b5-hint-glow` 脈動 2 次），語音播報「還剩X元，可以加選Y」。

### 14.4 語音系統設計

**開場語音**：「第[N]關，派對預算[M]元，必備：蛋糕和飲料，請在預算內採購派對用品！」

**商品選取語音**：選擇→「[商品名]，[金額]元」；取消→「取消[商品名]」

**足額語音（首次達到合法金額）**：「選好了，可以確認採購！」（搭配確認鈕 ready 脈動）

**超支語音**：「超過預算了！需要移除一些商品。」

### 14.5 視覺設計特色

| 元素 | 設計 | 說明 |
|------|------|------|
| 預算儀表條 | 顏色漸變 + 百分比標籤 | 即時視覺化預算壓力 |
| 必買商品卡 | 灰底鎖定（pointer-events:none）| 視覺告知「不可更動」|
| 確認鈕脈動 | 綠色光暈縮放（`b5ConfirmPulse`）| 引導下一步行動 |

## 十九、必買/選購分組佈局（2026-03-31，Round 42）

### 背景

B5 原本所有商品放在一個平鋪 grid 中，學生必須靠 `🔒 必買` 徽章來區分必買與選購商品。認知負擔較高。參照 **A4 商品分類 pattern**（不同商店類型使用不同卡片分組），將商品分為兩個視覺區段，讓「必買固定費用」與「可用選購預算」一目了然。

### 技術實作

**JS 修改**（`_renderRoundHTML`）：

```javascript
// 分拆為必買 / 選購兩組
const renderCard = item => `<div class="b5-item-card ..." data-id="${item.id}">...</div>`;
const mustItems   = g.items.filter(i => i.must);
const optItems    = g.items.filter(i => !i.must);
const mustTotal   = mustItems.reduce((s, i) => s + i.price, 0);
const optBudget   = Math.max(0, g.budget - mustTotal);

// HTML 結構：兩個 b5-section-group
<div class="b5-section-group">
    <div class="b5-section-hd b5-section-hd-must">
        <span>🔒 必買商品</span>
        <span class="b5-section-sub">${mustTotal} 元（固定）</span>
    </div>
    <div class="b5-items-grid">${mustItemsHTML}</div>
</div>
<div class="b5-section-group">
    <div class="b5-section-hd b5-section-hd-opt">
        <span>✨ 選購商品</span>
        <span class="b5-section-sub" id="b5-opt-budget">可用 ${optBudget} 元</span>
    </div>
    <div class="b5-items-grid" id="b5-items-grid">${optItemsHTML}</div>
</div>
```

**JS 修改**（`_updateTotalBar`）：即時更新 `#b5-opt-budget`：
```javascript
const optBudget = Math.max(0, g.budget - mustTotal);
const optSpent  = total - mustTotal;
const optRem    = optBudget - Math.max(0, optSpent);
optBudgetEl.textContent = optRem >= 0 ? `可用 ${optBudget} 元（剩 ${optRem}）` : `超出 ${-optRem} 元`;
optBudgetEl.style.color = optRem < 0 ? '#dc2626' : '';
```

**CSS 新增**（`b5_party_budget.css`）：
```css
.b5-section-hd-must { background: 琥珀漸層; border-left: 5px solid #f59e0b; }
.b5-section-hd-opt  { background: 翠綠漸層; border-left: 5px solid #10b981; }
```

### 教學設計

| 區段 | 顏色 | 功能 |
|------|------|------|
| 🔒 必買商品 | 琥珀/黃色 | 固定成本，無法不選，顯示小計 |
| ✨ 選購商品 | 翠綠色 | 彈性預算，顯示可用金額+即時剩餘 |

- 琥珀色（警示色）對應「必要支出」心理框架
- 翠綠色（機會色）對應「彈性選擇」心理框架  
- `b5-opt-budget` 即時更新讓學生清楚感知「我還能花多少」
- `_bindRoundEvents` / `_showBudgetHint` / `_updateTotalBar` 均使用 class 選擇器，不受 HTML 結構分組影響

### 搜尋關鍵字

- `b5-section-group`、`b5-section-hd-must`、`b5-section-hd-opt`、`b5-opt-budget`

## 二十、Round 43 困難模式翻牌動畫（2026-03-31）

### 功能說明
困難模式商品卡片首次點擊揭示價格時，由原先直接移除 `b5-price-hidden` class，改為先加入 `b5-flip-reveal` 動畫 class（`scaleX` 0.33s 動畫）。在動畫中點（150ms）時更新價格文字；330ms 後移除動畫 class。視覺效果為卡片「翻面」，翻面前顯示 `???`，翻面後顯示真實價格。

### 教學設計
- **參照模式**：C5 差額圖示漸進揭示 + A1/A2 coinFirst 服務亮燈效果
- **學習強化**：翻牌動畫製造「揭示感」，強化「原本未知→現在揭示」的認知轉換；學生更容易記住剛揭示的價格

### 關鍵實作
```javascript
// 困難模式翻牌（Round 43）
if (card.classList.contains('b5-price-hidden')) {
    card.classList.add('b5-flip-reveal');
    Game.TimerManager.setTimeout(() => {
        card.classList.remove('b5-price-hidden');
        const priceEl = card.querySelector('.b5-item-price');
        if (priceEl) priceEl.textContent = priceEl.dataset.price + ' 元';
    }, 150, 'ui');
    Game.TimerManager.setTimeout(() => card.classList.remove('b5-flip-reveal'), 330, 'ui');
    return;
}
```

```css
@keyframes b5FlipReveal {
    0%   { transform: scaleX(1);  opacity: 1; }
    40%  { transform: scaleX(0);  opacity: 0.4; }
    60%  { transform: scaleX(0);  opacity: 0.4; }
    100% { transform: scaleX(1);  opacity: 1; }
}
.b5-flip-reveal { animation: b5FlipReveal 0.33s ease; transform-origin: center; }
```

### 搜尋關鍵字
`b5-flip-reveal`、`b5FlipReveal`、`翻牌`
