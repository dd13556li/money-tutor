● Google NotebookLM × Claude Code 結合使用說明

  ---
  一、什麼是 NotebookLM

  Google NotebookLM（https://notebooklm.google.com）是 Google 推出的 AI 筆記工具，底層使用
   Gemini。核心概念是「以你的文件為基礎回答問題」，回答會標示來源段落，不會憑空捏造。

  ---
  二、安裝 / 設定方式

  完全不需要安裝，純網頁服務：

  1. 開啟 https://notebooklm.google.com
  2. 登入 Google 帳號
  3. 建立新 Notebook
  4. 上傳來源文件（支援格式如下）

  ┌──────────────────────┬─────────────────────┐
  │       來源類型       │        說明         │
  ├──────────────────────┼─────────────────────┤
  │ Google Docs / Slides │ 直接從雲端硬碟選取  │
  ├──────────────────────┼─────────────────────┤
  │ PDF                  │ 上傳本機 PDF        │
  ├──────────────────────┼─────────────────────┤
  │ 純文字 / Markdown    │ 上傳 .txt、.md 檔案 │
  ├──────────────────────┼─────────────────────┤
  │ 網址                 │ 貼上網頁 URL        │
  ├──────────────────────┼─────────────────────┤
  │ YouTube 影片         │ 貼上 YouTube 連結   │
  └──────────────────────┴─────────────────────┘

  針對本專案的做法：
  - 將 CLAUDE.md、report/A3_Unit_Completion_Report.md 等 Markdown 檔案直接上傳（另存為
  .txt 或直接上傳 .md）
  - 免費版最多 50 個來源 / 每個 Notebook

  四、會比較省 Token 嗎？

  是，但適用場景不同：

  Claude Code token 消耗情境：
    讀取大檔案（CLAUDE.md 629行、report 1900行）→ 每次都消耗 context

  NotebookLM 分流的情境：
    「A3 有哪些已修復的 Bug？」→ 問 NotebookLM，不消耗 Claude token
    「A4 的 optimalPaymentTargets 是什麼邏輯？」→ 問 NotebookLM

  仍需 Claude Code 的情境：
    實際修改程式碼、讀取即時檔案狀態、跨檔案追蹤 Bug

  實際省法：把「查資料、回顧歷史修復紀錄、理解架構」移到 NotebookLM，「動手改程式」留給
  Claude Code，可降低每次對話的 context 消耗。

  五、政策合規性

  Google 政策

  - 免費版：Google 可能使用上傳內容改善服務（需確認隱私設定）
  - 建議：登入後 → 設定 → 關閉「幫助改善 NotebookLM」選項
  - 若內容含敏感/商業機密程式碼，需評估是否符合公司資安政策

  Anthropic（Claude）政策

  - NotebookLM 與 Claude 完全獨立，不存在資料共享
  - 本專案的 report 檔案上傳 NotebookLM 不影響 Claude 的使用條款
  - 符合 Anthropic 政策（因為跟 Anthropic 沒有直接關係）

  七、輸出成檔案

  NotebookLM 的匯出選項如下：

  ┌──────────────────────┬───────────────────────────────────────────┐
  │         功能         │                 輸出方式                  │
  ├──────────────────────┼───────────────────────────────────────────┤
  │ 筆記（Notes）        │ 儲存為 Google Docs → 下載為 .docx 或 .pdf │
  ├──────────────────────┼───────────────────────────────────────────┤
  │ 自動生成的摘要 / FAQ │ 複製貼上 → 存成 .md 或 .txt               │
  ├──────────────────────┼───────────────────────────────────────────┤
  │ Audio Overview       │ 下載為 .mp3 音訊檔                        │
  ├──────────────────────┼───────────────────────────────────────────┤
  │ Mind Map（思維導圖） │ 目前僅限瀏覽，無直接匯出（可截圖）        │
  └──────────────────────┴───────────────────────────────────────────┘

  無法直接匯出為 Markdown，需自行複製後貼入編輯器存檔。

  ---
  八、建議工作流程

  修復 Bug → Claude Code 修改程式碼 + 更新 report/*.md
                      ↓
           定期（每週或里程碑後）將 CLAUDE.md + 所有 report
           重新上傳至 NotebookLM（覆蓋舊版本）
                      ↓
      需要查詢歷史記錄 / 理解架構 → 問 NotebookLM（省 token）
      需要實際修改程式碼           → 用 Claude Code

  ---
  總結：NotebookLM 適合作為「專案知識庫的查詢介面」，與 Claude Code
  互補而非取代。免費版對本專案完全夠用，主要限制是需手動同步文件，以及無法直接操作程式碼。
