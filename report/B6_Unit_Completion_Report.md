## 二、單元特色

### 2.1 教學定位

B6 是 B 系列中**流程最完整、最接近 A 系列深度**的單元，模擬完整的市場採購交易：
採購清單確認 → 多攤位採購 → 付款 → 找零。
學習路徑：**B5 預算規劃（選購）→ B6 市場採購（完整交易循環）→ A 系列（各場所模擬）**

### 2.2 三攤位系統

| 攤位 | 識別 | 商品數 |
|------|------|-------|
| 蔬菜攤 🥦 | `vegetable` | 6 件 |
| 水果攤 🍎 | `fruit` | 6 件 |
| 雜貨攤 🛒 | `grocery` | 6 件 |

學生需**切換標籤**在不同攤位間採購，符合真實菜市場場景。

### 2.3 三段式流程（對標 A 系列步驟化設計）

```
Phase 1: Shopping（採購）
    → 切換攤位標籤，點擊清單上的商品
    → 商品勾選後在清單上顯示 ✓

Phase 2: Payment（付款）
    → 點選鈔票/硬幣面額，累積付款金額
    → 達到消費總額時解鎖「確認付款」

Phase 3: Change（找零）
    → 顯示找零金額與面額組成
    → 語音播報找零金額
```

### 2.4 錯誤商品容錯設計

點到**不在清單**的商品：
- 浮動提示框：「❌ {商品名}不在今天的購物清單上」（2 秒後自動消失）
- 語音提示
- 播放 error.mp3
- **不跳題**（允許繼續採購，培養自我修正能力）

這與 A4 超市的「錯誤商品」處理一致，體現容錯設計哲學。

### 2.5 三種難度模式

| 維度 | 簡單 | 普通 | 困難 |
|------|------|------|------|
| 使用攤位數 | 1~2 | 2~3 | 3 |
| 購物清單件數 | 2 件 | 3~4 件 | 4~5 件 |
| 總消費金額 | 低（50~90 元）| 中（100~175 元）| 高（155~325 元）|
| 付款預算 | 整額或小額 | 多面額組合 | 大額多面額 |
| 找零金額 | 中等（35~110 元）| 中等（80~145 元）| 較多（45~200 元）|

### 2.6 正確商品彈出價格動畫（`_showPricePopup`）—— 2026-03-25 新增

**靈感來源**：A4 交易摘要中商品加入購物車的視覺回饋；A3 點餐確認動畫。

找到清單商品後觸發雙重視覺回饋：

1. **清單項目彈跳**（`.b6-list-bounce`）：列表中該項目輕彈 0.5s，強調「找到了！」
2. **浮動綠色價格標籤**（`.b6-price-popup`）：列表項目右側出現「+X 元」綠色標籤，向上漂浮後消失（`b6PriceFloat` 1.0s）

**設計目標**：
- 視覺確認「這個商品已加入購物籃」
- 讓學生看到每項商品的金額，邊買邊心算總金額
- 搭配現有的 `b6-basket-total` 更新，形成「點擊 → 看到金額 → 累計總價」的完整回饋迴路

**技術細節**：
- 使用 `position: fixed` + `getBoundingClientRect()` 定位（相對視窗，不受滾動影響）
- `Math.min(rect.right + 4, window.innerWidth - 80)` 防止標籤超出螢幕邊界
- `TimerManager.setTimeout` 1100ms 後自動移除 DOM 節點

### 2.7 攤位分頁需求件數徽章 —— 2026-03-25 新增

每個攤位分頁按鈕右上角顯示徽章，即時反映該攤位還有幾件任務商品待收集。

**狀態邏輯**（`_renderShoppingUI`）：
```javascript
const stallNeedIds  = mission.items.filter(i => i.stall === k).map(i => i.id);
const remaining     = stallNeedIds.filter(id => !g.collectedIds.has(id)).length;
// remaining > 0 → 紅色數字；remaining === 0 && stallNeedIds.length > 0 → 綠色 ✓
```

| 徽章狀態 | 樣式 | 語意 |
|---------|------|------|
| 紅色數字（e.g. `2`） | `.b6-stall-badge` | 此攤位有 N 件待收集 |
| 綠色 ✓ | `.b6-stall-badge-done` | 此攤位任務全完成 |
| 無徽章 | — | 此攤位本關無任務商品 |

**設計目標**：
- 不用看購物清單，直接從分頁就能知道「去哪個攤位」
- 全綠後一目瞭然知道可以結帳
- 每次切換攤位（`_renderShoppingUI` 重繪）自動更新徽章

**靈感來源**：C5「夠不夠」中用紅/綠指示燈快速傳遞「是/否」狀態 — 攤位徽章以計數擴展同一概念。

**CSS**（`b6_market_shopping.css`）：`.b6-stall-badge`（紅）、`.b6-stall-badge-done`（綠覆寫）

## 五、測驗流程

```
[設定] 難度 × 關卡數 × 重試模式
    ↓
[關卡開始] 語音：「今天要去菜市場，請按照清單買東西」
    ↓
[Phase 1 — Shopping]
    ├── 左側：購物清單（帶數量/單位）
    ├── 右側：攤位標籤 + 商品格（含名稱/單位/價格）
    ├── 點選攤位標籤 → 切換顯示攤位商品
    ├── 點選清單商品 → 勾選（✓），清單對應項目標記
    ├── 點選非清單商品 → 浮動錯誤提示 + 語音
    └── 清單全勾 → 自動進入 Phase 2
    ↓
[Phase 2 — Payment]
    ├── 顯示：消費總金額、面額按鈕（7 種）
    ├── 點選面額 → 累積付款金額，即時顯示已付/待付
    ├── 超額付款 → 語音警告
    └── 等於或超過（整額）→ 解鎖「確認付款」
    ↓
[Phase 3 — Change]
    ├── 顯示找零金額與面額組成
    ├── 語音：「找你 X 元」
    └── 「下一關」→ 進下一題 or showResults()
```

### 7.1 付款最佳面額提示（`_showPaymentHint`）（2026-03-25）

**靈感來源**：B1 `_calcOptimalCoins` / `_showCoinHint`（最佳硬幣組合高亮）；A4 付款提示勾勾。

簡單 / 普通模式付款畫面新增「💡 提示」按鈕（困難模式隱藏）。點擊後：

1. **貪婪演算法**（`_showPaymentHint`）：由大面額到小，計算恰好湊到 `total` 的最少面額組合
2. **面額按鈕高亮**：相關按鈕加入 `.b6-bill-hint`（黃色描框 + 脈動動畫）
3. **提示訊息**：頁面下方顯示「💡 建議付法：百元 × 1 + 五元 × 1 = 105 元」
4. **語音**：「建議付百元，再加五元」
5. 6 秒後自動移除高亮與訊息

**技術實作**：
- HTML：`${difficulty !== 'hard' ? '<button class="b6-hint-btn"…>' : ''}` 條件渲染
- CSS：`.b6-hint-btn`（黃色）、`.b6-bill-hint`（黃框脈動）、`.b6-hint-toast`（黃色訊息卡）

### 13.2 困難模式找零驗算（✅ 2026-03-24 實作完成）

困難模式付款後，新增三選一找零驗算步驟。

**實作架構**：`_showChange()` → 困難模式路由至 `_showChangeQuiz()` → 完成後呼叫 `_showChangeResult()`

**`_showChangeQuiz(paid, total, change)`**：
- 語音：「你付了X元，買菜共花了Y元，應該找回多少元？」
- 產生 2 個干擾項（±5/10/15/20/25/30 偏移，確保 > 0）
- 三個按鈕：`.b6-change-opt`（答對→`.b6-change-opt-correct`，答錯→`.b6-change-opt-wrong`）
- retry 模式：錯誤選項 disabled，可繼續作答
- proceed 模式：全部 disabled，顯示正確答案，1400ms 後進入結果頁

**`_showChangeResult(paid, change)`**：
- `g.correctCount++`（從 `_showChange` 移至此處，確保 quiz 流程也計分）
- 顯示 🎉 找零金額 + 下一關按鈕

**CSS 新增**（`css/b6_market_shopping.css`）：
`.b6-change-question`、`.b6-change-opts`、`.b6-change-opt`、`.b6-change-opt-correct`、`.b6-change-opt-wrong`

**計算邏輯**：
- `paid` = 最小可覆蓋 `totalPrice` 的整數面額（例：買 340 元 → 付 500 元）
- `change` = `paid - totalPrice`（例：500 - 340 = 160 元）
- 答對 → 顯示「找你 160 元」→ 關卡完成
- 答錯（retry）→ 重新輸入；答錯（proceed）→ 顯示正確答案 → 下一關

## 十八、教學設計詳析（參照 F/C/A 格式，2026-03-29）

### 18.1 教學定位與學習目標

**核心定位**：B6 是 B 系列中「流程最完整、整合度最高」的單元，以菜市場購物模擬完整的「採購 → 付款 → 找零」三段流程，整合 B 系列其他單元的知識並對應 A 系列的生活場景操作能力。

**學習目標層次**：
1. **知識層**：了解市場採購流程，認識各攤位商品分類
2. **技能層**：閱讀採購清單，累加多商品金額，計算付款與找零
3. **應用層**：完整執行真實購物情境，從「知道要買什麼」到「完成付款找回零錢」

**與其他單元的銜接**：
- 前置：B1（付款計算）→ B2（加減運算）→ B4（比較判斷）→ C6（找零）
- B6 是 B 系列的「實戰整合關卡」，整合所有先備知識於完整情境

### 18.2 測驗內容設計

**三攤位分類設計理念**：市場分攤模擬真實購物空間，學生需要「移動」到不同攤位找商品，訓練情境空間感與清單管理能力。

**任務清單設計規則**：
- 每關指定 2~5 件商品，分布於 1~3 個攤位
- 商品選取確保有明確的「攤位歸屬」，避免混淆
- 困難模式商品橫跨全部 3 攤位，需要完整的攤位巡迴

**找零三選一設計（困難）**：
- 正確答案 ±10~50 元偏移生成 2 個干擾項
- 干擾項確保均為正整數，且與正確答案視覺上相近（防猜測）
- 允許「答錯後繼續選」（proceed 模式），不懲罰摸索

### 18.3 互動方式詳析

**完整三段式流程**：
```
[採購段]
開場任務彈窗 2.8s → 分頁切換攤位 → 點擊清單商品
  ↳ 點到不在清單的商品 → 錯誤 toast + 提示「這裡需要：X」（容錯不扣分）
  ↳ 攤位全收 → 徽章變綠 ✓

[找零段]

hard：三選一問答 → 轉場卡（非最後關）/ 完成畫面
```

**錯誤容錯設計哲學**：採購段點錯商品「只提示，不扣分」，強調「學習如何找到正確商品」而非懲罰錯誤。

## 二十三、找零計算輔助面板（2026-03-30）

### 背景

C6 單元具備內建計算輔助工具（可展開的算式提示），B6 找零測驗（`_showChangeQuiz`）答題時，學生需要自行計算 `付款金額 − 商品總金額`，難度較高。

### 實作

在 `_showChangeQuiz(paid, total, change)` 的選項區塊上方插入「🧮 幫我算一算」切換按鈕與可摺疊面板：

```
付了   X 元
− 花了 Y 元
──────────
= 找零 ？元
```

**HTML 結構**：`.b6-calc-toggle` 按鈕 + `.b6-calc-panel`（預設隱藏，`.visible` 顯示）

**事件綁定**：`Game.EventManager.on(calcToggle, 'click', fn, {}, 'gameUI')`，切換時改按鈕文字為「🙈 收起計算」/ 「🧮 幫我算一算」。

### CSS 類別

| 類別 | 用途 |
|------|------|
| `.b6-calc-toggle` | 切換按鈕（黃橙色漸層） |
| `.b6-calc-panel` | 摺疊面板（預設 `display:none`） |
| `.b6-calc-panel.visible` | 展開狀態 |
| `.b6-cp-row` | 每列容器 |
| `.b6-cp-label` | 左側說明文字（「付了」/「花了」/「找零」） |
| `.b6-cp-num` | 金額數字（橙色大字） |
| `.b6-cp-unit` | 「元」單位 |
| `.b6-cp-op` | 運算符號（`−`） |
| `.b6-cp-sub` | 被減數行容器 |
| `.b6-cp-divider` | 橫線分隔 |
| `.b6-cp-ans` | 答案行容器 |
| `.b6-cp-q` | 問號佔位（`？`） |

### 教學設計

- 遵循 **C6 計算輔助 pattern**：可隱藏，不強制顯示
- 困難度降低：學生可自選是否使用輔助
- 直式算式格式符合小學數學教學規範

## 十六、Round 43 付款提示面額色塊（2026-03-31）

### 功能說明
`_showPaymentHint` 的提示 toast 由純文字「千元 × 1 + 百元 × 2 = 1200 元」改為彩色面額色塊（`.b6-hint-badge`）。每個面額以對應顏色（來自 `B6_BILLS[].color`）的圓角標籤顯示，搭配 CSS 自訂屬性 `--bc` 動態注入顏色。

### 教學設計
- **參照模式**：C4/C6 最佳付款面額高亮 + B1 `_showCoinHint` 面額圖示卡（Round 37）
- **學習強化**：顏色編碼幫助學生快速辨識面額大小（紫色千元、棕色五百、藍色百元等），視覺記憶比純文字更持久

### 關鍵實作
```javascript
const badgesHTML = B6_BILLS
    .filter(b => used[b.value])
    .map(b => `<span class="b6-hint-badge" style="--bc:${b.color}">${b.value}元×${used[b.value]}</span>`)
    .join('<span class="b6-hint-plus">+</span>');
toast.innerHTML = `<strong>💡 建議付法</strong>
    <div class="b6-hint-badges">${badgesHTML}<span class="b6-hint-eq">= ${total}元</span></div>`;
```

```css
.b6-hint-badge {
    background: var(--bc, #374151);
    color: #fff;
    border-radius: 14px;
    padding: 4px 10px;
    font-weight: 700;
}
```
