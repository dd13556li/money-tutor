## 二、單元特色

### 2.1 教學定位

B2 訓練「閱讀記帳日記、追蹤收支變化、計算最終餘額」的能力。
學習路徑：**加減法（F 系列）→ 貨幣金額（C 系列）→ 收支記錄（B2）→ 預算規劃（B5）**

### 2.2 日記卡格式設計

每題呈現一週的收支日記，格式設計仿照真實記帳本：

```
期初餘額：100 元
▲ + 媽媽給零用錢  💰  +50 元
▼ - 買飲料        🧋  -30 元
────────────────────────
剩餘：? 元
```

視覺設計：
- `income`（收入）：綠色 ▲ + 圖示
- `expense`（支出）：紅色 ▼ - 圖示
- 每列含情境 emoji、事件名稱、金額

### 2.3 三種難度模式

| 模式 | 事件筆數 | 題庫組數 | 作答方式 | 干擾項設計 |
|------|---------|---------|---------|---------|
| 簡單 | 2 筆 | 12 組 | 三選一選擇題 | 間距 ≥20 元（fixed delta array）|
| 普通 | 4 筆 | 12 組 | 數字鍵盤輸入 | — |
| 困難 | 6 筆 | 12 組 | 數字鍵盤輸入 | 含正負混合，大額交叉 |

**簡單模式干擾項設計**（對照 C1 動態選項數量精神）：
固定使用 delta array 而非隨機偏移，確保三個選項間距均 ≥20 元，避免無效干擾：
```javascript
const deltas = [-40, -20, +20, +40];
// 隨機取兩個，確保不等於正確答案且非負數
```

### 2.4 重試 / 繼續模式

| 模式 | 行為 |
|------|------|
| `retry`（預設）| 答錯後語音「不對喔，再試一次」→ 重新啟用選項/鍵盤，可繼續作答 |
| `proceed` | 答錯後語音「正確答案是 X 元」→ 2500ms 後自動跳下一題 |

### 2.5 計算過程提示（`_showCalcBreakdown`）（2026-03-25）

**靈感來源**：C 系列在答錯後顯示計算過程的設計哲學（C4 分步驟展示、C6 找零明細）。

普通 / 困難模式（數字鍵盤）答錯時，立即在鍵盤下方顯示逐步計算過程：

```
💡 計算過程
起  開始有      100 元
＋  媽媽給零用錢  50 元
－  買飲料        30 元
＝  結果         120 元
```

**設計重點**：
- 答錯即顯示（不需等第二次），讓學生能立即對照修正
- `retry` 模式：計算表保留在畫面，學生可參考後重新輸入
- `proceed` 模式：計算表顯示後 2.5 秒跳題（比原 2.2 秒略長，給足閱讀時間）
- 防重複守衛：`document.querySelector('.b2-calc-breakdown')` 確保每題只顯示一次
- CSS：`.b2-calc-breakdown`（黃色背景卡）、`.b2-bd-row`、`.b2-bd-op.income`（綠）、`.b2-bd-op.expense`（紅）

### 2.6 Easy 模式逐項動畫高亮（`_animateEasyEntries`）—— 2026-03-25 新增

**靈感來源**：C2 數錢「逐一點擊計數」互動——學生逐個點擊硬幣，累加金額；F5 普通模式點擊計數互動。

B2 easy 模式的教學目標是「看日記條目，選出正確餘額」。新增動畫後：

1. 題目渲染後，**選項區（answer area）先隱藏**
2. 日記事件行**逐項高亮**（每 800ms 一項）：黃底 + 邊框 + 微縮放
3. 全部高亮完成後，**500ms 後選項淡入出現**

**教學效果**：
- 引導學生按順序閱讀每一筆收支（而非一眼掃完就猜答案）
- 視覺節奏對應「一項一項加減」的心算過程
- 選項晚出現 → 確保學生先思考，再選答案

**技術細節**：
- `visibility: hidden`（不是 `display:none`）保留版面空間，避免 layout shift
- `TimerManager.setTimeout` 管理全部計時，切換題目時自動清除（`clearAll` in `renderQuestion`）
- CSS：`.b2-entry-dim`（opacity 0.25）/ `.b2-entry-active`（黃底 + `box-shadow: 0 0 0 2px #fde047`）

### 2.7 Easy 模式逐項小計顯示（`b2-running-total`）—— 2026-03-25 新增

在 `_animateEasyEntries` 每個事件行高亮的同時，日記卡片「開始有 X 元」下方插入**即時小計顯示卡**：

```
目前小計   [120 元]   ← 綠色（收入後）
目前小計   [90 元]    ← 紅色（支出後）
```

**設計邏輯**：
- `_animateEasyEntries` 開頭預先計算所有步驟餘額（`balances[]` 陣列）
- 每步高亮時更新 `#b2-rt-val` 文字，並切換 `b2-rt-up`（綠）/ `b2-rt-down`（紅）class
- 觸發 `void offsetWidth` 強制 reflow 後重設 `b2RtPop` 彈出動畫，每步都有彈跳視覺
- 動畫結束後（選項淡入時）自動移除小計卡（`runEl.remove()`）

**靈感來源**：F5 量比較視覺化橫條 — 幫助視覺型學習者直觀感受「數字在每次操作後如何變化」，不必心算全部加減再回溯。

**教學效果**：
- 學生能即時看到「收入讓數字變大（綠）、支出讓數字變小（紅）」
- 搭配逐項動畫節奏，形成「看事件 → 看數字變化」的完整回饋迴路
- 最終選項出現時，學生已在腦中形成正確答案的預期

**CSS**（`b2_allowance_diary.css`）：`.b2-running-total`（藍框卡）、`.b2-rt-label`、`.b2-rt-val`、`.b2-rt-up`（綠）、`.b2-rt-down`（紅）、`@keyframes b2RtPop`

## 五、測驗流程（對照 C1 答題流程格式）

```
[初始化] → showSettings()
    ↓
[設定頁] 選擇：難度 × 題數 × 重試模式
    ↓（全選後 startBtn 解鎖）
[題目頁] renderQuestion()
    ├── 渲染日記卡（期初 + 事件列表）
    ├── 播報語音「請看日記…」
    └── 渲染作答區（選項 or 數字鍵盤）
         ↓
    [作答]
    ├─ 正確 → 播「答對了！」→ 350ms → 進下一題（或 showResults）
    └─ 錯誤（retry）→ 播「不對喔…」→ 重新啟用
           （proceed）→ 播「正確答案…」→ 1400ms → 進下一題
         ↓
[完成] showResults() — 統一 C/F 完成畫面樣式
```

## 七、技術注意事項

- `B2_TEMPLATES` 為靜態陣列，不動態生成，因此每難度題庫上限固定（12/12/12 組，共 36 組）
- **2026-03-24**：三種難度全部擴充至 12 組（easy 8→12、normal 8→12、hard 6→12）
- 超出題數時**循環使用**（模池機制），各難度 ≤12 題時不重複，20 題設定下最多重複 8 題
- 數字鍵盤最多 5 位數（上限 99999 元），超出時不接受輸入
- 簡單模式干擾項使用 fixed delta，確保不出現負數選項（若 `answer + delta < 0` 則過濾）

## 九、跨系列特色分析（2026-03-24）

### 9.1 B2 vs F6 數的組成（同為「加減累算」類型）

| 特性 | B2 零用錢日記 | F6 數的組成 |
|------|------------|-----------|
| 核心運算 | 多步正負加減（收入+/支出-）| 分解/合成（拆解一個數為兩數）|
| 呈現方式 | 日記敘事（情境豐富）| 數字符號（抽象）|
| 資料庫 | 固定 28 組模板（8+8+12，2026-03-24 擴充）| 動態生成（無限）|
| 答題方式 | 三選一/鍵盤輸入 | 拖曳/點擊 |
| 情境模態 | 文字敘述（有 icon）| 純數字/圖形 |

### 9.2 B2 vs C2 數錢（同為「累加計算」概念）

| 特性 | B2 | C2 |
|------|----|----|
| 累加對象 | 生活事件金額（正負）| 錢幣面額（正值）|
| 互動方式 | 讀完後直接輸入答案 | 點擊計數（C2 普通：逐一計數）|
| 難度模態 | 事件數量 + 輸入方式 | 感知模態（點擊→視覺→聽覺）|
| 情境 | 生活記帳（抽象）| 真實錢幣（具象）|

**啟示**：C2 的「點擊計數」互動（學生逐一點擊事件來累加金額）可引入 B2 普通模式——讓學生點擊每筆收支事件，系統累加顯示當前餘額，再輸入最終答案。這能降低心算負擔，符合特殊需求學習者。

## 十、觸控與桌面支援（2026-03-27 補充）

### 10.1 互動模式與觸控

B2 以「閱讀 → 輸入」為主要互動，無拖曳操作，觸控支援較簡單。

| 互動 | 桌面 | 行動 |
|------|------|------|
| 選擇答案（easy）| 點擊選項卡片 | 觸控點擊 |
| 輸入金額（normal/hard）| 鍵盤輸入 → Enter 確認 | 數字鍵盤 / 系統軟鍵盤彈出 |
| 滾動日記事件 | 滑鼠滾輪 | 手指滑動 |
| 提示觸發 | 點擊 💡 鈕 | 觸控 💡 鈕 |

**easy 模式逐項動畫（B2 特有）**：

## 十三、教學設計詳析（參照 F/C/A 格式，2026-03-29）

### 13.1 教學定位與學習目標

**核心定位**：B2 透過「閱讀週記帳本」的情境，訓練學生執行多步驟連續加減法，並理解「收入使餘額增加、支出使餘額減少」的記帳概念。

**學習目標層次**：
1. **知識層**：理解收入（+）與支出（−）的差異，認識帳本格式
2. **技能層**：從起始金額出發，依序執行加減法，算出最終餘額
3. **應用層**：能閱讀真實記帳記錄，評估一週的收支狀況

### 13.2 測驗內容設計

**日記事件選材**：選自學生日常生活（零用錢、外出消費、家事獎勵、節慶禮金），每個事件附明確說明文字幫助閱讀理解。

**選項生成策略（簡單模式）**：
- 正確答案 1 個，干擾項差距 20~50 元（保證可辨識）

**簡單模式（視覺動畫 + 選擇）**：逐項動畫引導學生跟著「數一數」，降低認知負荷；即時小計讓學生直觀感受金額如何變動；選擇題聚焦於理解加減方向。

**普通 / 困難模式（閱讀 + 鍵盤）**：全部條目一次顯示，訓練學生自主整理資訊；鍵盤輸入訓練數字書寫；逐步算式提示（答錯後）提供「正確計算過程」，而非直接給答案。

## 十二、簡單模式事件卡片視覺強化（2026-03-31，Round 41）

### 背景

B2 簡單模式的事件列（`.b2-event-row`）原為細字排列，視覺密度高，對認知負擔較重的學生較難區分收入與支出。參照 **F1 視覺配對模式**（大色塊圖示）與 **A4 商品卡片**（彩色邊框）設計語言，為簡單模式增加卡片視覺。

### 技術實作

**JS 修改**（`_renderQuestionHTML`）：

1. 在 `.b2-diary` 加入 `data-diff="${diff}"` 屬性，讓 CSS 可依難度套用不同樣式。
2. 在 `.b2-event-row` 加入 `${e.type}` class（`income` 或 `expense`），讓 CSS 精準選取。

```javascript
// Before
<div class="b2-diary">
<div class="b2-event-row" style="animation-delay:...">

// After
<div class="b2-diary" data-diff="${diff}">
<div class="b2-event-row ${e.type}" style="animation-delay:...">
```

**CSS 新增**（`b2_allowance_diary.css` 末尾）：

```css
/* ── Round 41：B2 簡單模式事件卡片視覺強化 ── */
.b2-diary[data-diff="easy"] .b2-event-row {
    padding: 14px 18px;
    border-radius: 12px;
    border-bottom: none;
    margin-bottom: 8px;
    min-height: 60px;
    box-shadow: 0 2px 6px rgba(0,0,0,0.07);
}
.b2-diary[data-diff="easy"] .b2-event-row.income {
    background: linear-gradient(135deg, #f0fdf4, #dcfce7);
    border-left: 5px solid #22c55e;  /* 綠色 = 收入 */
}
.b2-diary[data-diff="easy"] .b2-event-row.expense {
    background: linear-gradient(135deg, #fef2f2, #fee2e2);
    border-left: 5px solid #ef4444;  /* 紅色 = 支出 */
}
.b2-diary[data-diff="easy"] .b2-event-icon { font-size: 28px; }
.b2-diary[data-diff="easy"] .b2-event-name { font-size: 17px; font-weight: 700; }
.b2-diary[data-diff="easy"] .b2-event-amount { font-size: 1.45rem; }

.b2-diary[data-diff="easy"] .b2-running-val { font-size: 12px; margin-left: auto; }
```

### 為何用 CSS 屬性選擇器而非 JS 條件渲染

- **零 HTML 結構差異**：普通/困難模式的 `_animateEasyEntries`、`_renderQuestionHTML` 不需分支
- **向後相容**：`_animateEasyEntries` 的 `.b2-entry-dim` / `.b2-entry-active` 操作不受影響，因為只額外加了 class，querySelector 仍抓到所有 `.b2-event-row`
- **CSS cascade 安全**：`[data-diff="easy"]` specificity 為 0-1-1（屬性+類別），高於原本的 0-0-1（只有類別），不需 `!important`

### 視覺設計

| 類型 | 背景 | 左邊框 | 意義 |
|------|------|--------|------|
| 收入（income） | 淡綠漸層 `#f0fdf4→#dcfce7` | 5px `#22c55e` 綠 | 錢進來了 |
| 支出（expense） | 淡紅漸層 `#fef2f2→#fee2e2` | 5px `#ef4444` 紅 | 錢出去了 |

- 普通/困難模式維持原本細字排列，不受影響（屬性選擇器 `[data-diff="easy"]` 精準隔離）
- 搜尋關鍵字：`data-diff`, `b2-event-row income`, `b2-event-row expense`

## 十三、Round 43 困難模式聽力記憶模糊（2026-03-31）

### 功能說明
困難模式新增「聽力記憶」體驗：日記事件的 icon 與名稱在初始狀態以 `blur(6px)` 模糊顯示，金額與收支徽章保持可見。學生需依賴語音重播記憶事件內容，再計算最終餘額。標題欄加入「👁️ 查看」按鈕，點擊後解除模糊（切換 `b2-revealed` class），同時變成「🙈 隱藏」。

### 教學設計
- **參照模式**：C1 困難模式音色識別（只憑聲音辨識錢幣種類）
- **學習強化**：強迫使用工作記憶（Working Memory）記住每筆事件，而非視覺掃描；訓練學生「先聽後算」的數學策略
- **降階備援**：「查看」按鈕提供安全網，避免挫折感過高

### 關鍵實作
```javascript
// _renderQuestionHTML：diary div 加 b2-memory-mode
<div class="b2-diary${diff === 'hard' ? ' b2-memory-mode' : ''}" data-diff="${diff}">

// 標題欄加入揭示按鈕
${diff === 'hard' ? `<button class="b2-reveal-btn" id="b2-reveal-btn">👁️ 查看</button>` : ''}

// _bindQuestionEvents：切換邏輯
const revealBtn = document.getElementById('b2-reveal-btn');
if (revealBtn) {
    Game.EventManager.on(revealBtn, 'click', () => {
        const revealed = diary.classList.toggle('b2-revealed');
        revealBtn.textContent = revealed ? '🙈 隱藏' : '👁️ 查看';
    }, {}, 'gameUI');
}
```

```css
.b2-diary.b2-memory-mode .b2-event-icon,
.b2-diary.b2-memory-mode .b2-event-name { filter: blur(6px); transition: filter 0.3s; }
.b2-diary.b2-memory-mode.b2-revealed .b2-event-icon,
.b2-diary.b2-memory-mode.b2-revealed .b2-event-name { filter: none; }
```

### 搜尋關鍵字
`b2-memory-mode`、`b2-reveal-btn`、`b2-revealed`

### 技術要點

- **CSS cascade 安全**：`[data-diff="easy"]` specificity 為 0-1-1（屬性+類別），高於原本 0-0-1（只有類別），不需 `!important`
- **聽力記憶模糊**：只模糊 icon 與名稱，金額與收支徽章保持可見；`b2-revealed` toggle 確保反覆揭示/隱藏均正常
- **主題動畫導引**：2400ms 觸發 = 任務彈窗（2200ms）關閉後，不干擾彈窗語音；隨機/未設定主題跳過，避免不相關提示
- 連勝徽章（B 系列統一）：`quiz.streak`；達 3/5 觸發 `_showStreakBadge(streak)`
